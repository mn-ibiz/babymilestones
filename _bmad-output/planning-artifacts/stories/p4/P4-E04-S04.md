# P4-E04-S04 — Daily dispatch report

**Status:** Ready for development
**Phase:** P4 — WooCommerce Sync + Events Ticketing
**Epic:** P4-E04 — POS ↔ WooCommerce Sync
**Estimate:** 1–3 days

## Job To Be Done

As shop ops, I want an end-of-day summary of online orders dispatched and still pending.

## Acceptance Criteria

- AC1: Report covers WooCommerce-originated orders only (in-store POS sales have their own end-of-day in P2-E04-S05).
- AC2: Counts by `local_status`, total value (KES), average pack time (new → ready), average dispatch time (ready → dispatched).
- AC3: CSV export.
- AC4: Date filter; defaults to today.
- AC5: Includes a "Sync health" row: orders with stuck/failed Woo writebacks in the dead-letter (link to S07 view).

## Technical Notes

- Reads from `wc_orders` + `order_events`. No live Woo call.

## Tests

- Unit: pack-time and dispatch-time calculations across edge cases (cancelled mid-flight, manually-reversed transitions).
