# P2-E03-S02 — Attendant check-in screen

**Status:** Ready for development
**Phase:** P2 — Bookings, Subscriptions, POS, Loyalty Redemption
**Epic:** P2-E03 — Pickup Authorisation & Free-Text Observations
**Estimate:** 1–3 days

## Job To Be Done

As an attendant (operated via Reception's screen), I want to check children in for a session in seconds.

## Acceptance Criteria

- AC1: Today's session slots listed; tap → booking list for that slot.
- AC2: For each booking: child card with name + photo (if consented) + drop-off time field.
- AC3: Check-in triggers wallet debit (P1-E03-S05) and records `attendance.checked_in_at`.
- AC4: Bulk check-in supported (rare but useful).

## Technical Notes

Reception screen sub-route. Same auth as Reception.
