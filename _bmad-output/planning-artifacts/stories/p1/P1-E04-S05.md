# P1-E04-S05 — Paystack webhook (signature + replay protection)

**Status:** Ready for development
**Phase:** P1 — Foundation + Wallet + Parent Account
**Epic:** P1-E04 — Payments Adapter
**Estimate:** 1–3 days

## Job To Be Done

As the system, I must trust Paystack webhooks cryptographically and accept each one only once.

## Acceptance Criteria

- AC1: `POST /webhooks/paystack` verifies `x-paystack-signature` via HMAC-SHA512 with secret; constant-time compare.
- AC2: Invalid signature → 401, no DB writes.
- AC3: `paystack_event.id UNIQUE`; replay → 200 OK, no work.
- AC4: `charge.success` event → `wallet.post(topup)`.

## Technical Notes

`packages/payments/paystack/verify.ts` with timing-safe compare.

## Tests

Tampered payload rejected; replay returns 200 without re-posting.
