# P1-E06-S01 — Configure float accounts (per till / per bank)

**Status:** Ready for development
**Phase:** P1 — Foundation + Wallet + Parent Account
**Epic:** P1-E06 — Treasury & Float Segregation
**Estimate:** 1–3 days

## Job To Be Done

As admin, I want to declare which accounts hold customer wallet float so the system can reconcile against them.

## Acceptance Criteria

- AC1: `float_accounts` table: name, kind (`mpesa_till` | `bank` | `cash_drawer`), opening balance, opening date.
- AC2: Admin CRUD with audit.
- AC3: Each top-up entry tags a `float_account_id` based on payment method.

## Technical Notes

Migration adds `wallet_ledger.float_account_id`. Backfill historical entries to "default" account at deploy time (will be empty in P1).
