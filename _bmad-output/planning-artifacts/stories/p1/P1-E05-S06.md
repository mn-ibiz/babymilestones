# P1-E05-S06 — Print + SMS-stub receipt from Reception

**Status:** Ready for development
**Phase:** P1 — Foundation + Wallet + Parent Account
**Epic:** P1-E05 — Reception Operator Surface *(heartbeat)*
**Estimate:** 1–3 days

## Job To Be Done

As Reception, I want to print or text a receipt to a parent after a transaction.

## Acceptance Criteria

- AC1: After any payment, a "Print" + "SMS" button pair appears.
- AC2: Print uses browser's default printer (Decision 13).
- AC3: SMS uses stub adapter (P1-E09).
- AC4: Reprint available from the transaction history at any time.

## Technical Notes

Print template uses `ReceiptPreview` compound from `packages/ui`.

## Dependencies

- P1-E08
- P1-E09.
