# P1-E07-S02 — Attribution role per service

**Status:** Ready for development
**Phase:** P1 — Foundation + Wallet + Parent Account
**Epic:** P1-E07 — Service Catalogue & Pricing
**Estimate:** 1–3 days

## Job To Be Done

As admin, I want each service to declare whether it needs a staff attribution slot (stylist for salon; instructor for talent; none for events).

## Acceptance Criteria

- AC1: `services.attribution_role` ENUM nullable.
- AC2: If non-null, Reception's booking flow forces a `staff` pick from that role's active members.
- AC3: If null, attribution is optional.
