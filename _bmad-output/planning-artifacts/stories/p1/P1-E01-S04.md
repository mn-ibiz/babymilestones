# P1-E01-S04 — SSO across subdomains

**Status:** Ready for development
**Phase:** P1 — Foundation + Wallet + Parent Account
**Epic:** P1-E01 — Identity & SSO Foundation
**Estimate:** 1–3 days

## Job To Be Done

As a signed-in user, I want my login to carry across the custom apps — `platform`, `pos`, `admin` — so I'm not re-prompted. (The standalone WooCommerce site is excluded — it has its own auth.)

## Acceptance Criteria

- AC1: Cookie domain `.babymilestones.co.ke` set on login from any app.
- AC2: All apps read session via the same middleware (`packages/auth/middleware.ts`).
- AC3: Logout from one app invalidates the session everywhere (Redis `DEL`).
- AC4: Role mismatch (e.g., parent landing on `admin.*`) → 403 with redirect to home.
- AC5: CSRF: double-submit cookie token required on POST/PUT/DELETE.

## Technical Notes

Opaque token in Redis (not JWT — role changes need instant invalidation). `session.touch()` extends TTL on each request.

## Tests

Open two browser tabs on different subdomains; logout in one invalidates the other within 5 seconds.
