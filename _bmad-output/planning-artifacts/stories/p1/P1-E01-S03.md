# P1-E01-S03 — Admin / Reception / Cashier login

**Status:** Ready for development
**Phase:** P1 — Foundation + Wallet + Parent Account
**Epic:** P1-E01 — Identity & SSO Foundation
**Estimate:** 1–3 days

## Job To Be Done

As an admin or staff user, I want to log into the admin app with my email + password so I can do my job.

## Acceptance Criteria

- AC1: Email + password (not phone+PIN) → session for staff role.
- AC2: Role determines landing page (`/reception`, `/treasury`, `/admin`).
- AC3: Password complexity enforced at creation: ≥10 chars, mixed.
- AC4: Same SSO cookie machinery as parents but on `admin.babymilestones.co.ke`.
- AC5: Audit log captures `auth.staff.login`.

## Technical Notes

`users.user_type ENUM('parent','staff')`. Staff have email; parents have phone. `packages/auth/staff.ts`.

## Tests

Cross-domain cookie issuance; staff cannot use parent flow and vice-versa.
